## Dependency notes

- Plans 001–005 are correctness and calibration prerequisites. Do not enable
  model authority before they land.
- Plan 002 precedes model planning because a planner is useless if several
  same-host operators collapse into one SOAR query.
- Plan 003 precedes Plans 004, 007, and 008 because model scores must not be
  exposed as calibrated probabilities.
- Plan 005 precedes Plan 006 so the planner receives executable, host-bound
  obligations rather than template IDs or evidence IDs masquerading as hosts.
- Plans 006–008 must launch in `shadow` mode and graduate independently.

## Recommended rollout

2. Land 002–005 and collect a fresh, leakage-free baseline.
3. Run 006–008 in shadow mode for at least one representative evaluation

4. Enable model influence only for slices with measured non-inferiority,

## Findings considered and rejected

- Replace all fixed thresholds with an LLM: rejected. Evidence integrity,
  permissions, budgets, probability calibration, and stopping safety require
  deterministic or statistically calibrated mechanisms.
- Let the model write SessionGraph directly: rejected. Models may propose
  typed intents; schema validation and K-phase code remain the only writers.
- Treat model self-reported confidence as probability: rejected. Confidence
  remains unavailable until calibrated against independent labels.
